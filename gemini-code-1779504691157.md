# 📄 제품 요구사항 문서 (PRD - Product Requirement Document)

## 1. 프로젝트 개요
* **프로젝트명**: 소규모 건물 관리비 체크 시스템
* **목적**: 9세대 거주 건물의 매월 고정 관리비(2만 원) 납부 현황을 투명하게 공유하고, 별도의 유료 API 비용 없이 입금 알림 및 확인 과정을 효율화하는 모바일 최적화 웹 서비스 구축
* **주요 타겟**: 건물 관리인(1명) 및 세대원(9세대)

---

## 2. 개발 테크 스택 (Tech Stack)
* **웹 서버 & 호스팅**: Vercel (Next.js App Router)
* **데이터베이스**: Supabase (PostgreSQL)
* **스타일링**: Tailwind CSS + Shadcn UI (모바일 반응형 컴포넌트 활용)
* **인증 구조**: 세대별 고정 비밀번호(4자리) 기반의 단순 세션/미들웨어 체크 인증

---

## 3. 데이터베이스(Supabase) 테이블 설계

### ① 세대 정보 테이블 (`households`)
| 컬럼명 (Column) | 타입 (Type) | 제약 조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `id` | UUID | PRIMARY KEY, DEFAULT gen_random_uuid() | 고유 식별자 |
| `room_no` | VARCHAR | UNIQUE, NOT NULL | 호수 (예: 'B01', '101') |
| `password` | VARCHAR | NOT NULL, DEFAULT '1234' | 접속용 비밀번호 (4자리) |
| `is_admin` | BOOLEAN | DEFAULT FALSE | 관리자 여부 (`true` 시 관리자 기능 권한) |

### ② 관리비 납부 현황 테이블 (`payments`)
| 컬럼명 (Column) | 타입 (Type) | 제약 조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, GENERATED ALWAYS AS IDENTITY | 고유 식별자 |
| `room_no` | VARCHAR | REFERENCES households(room_no) ON UPDATE CASCADE | 납부 세대 호수 |
| `payment_month` | VARCHAR(7) | NOT NULL | 대상 년/월 (예: '2026-05') |
| `paid_date` | DATE | NOT NULL, DEFAULT CURRENT_DATE | 실제 이체 날짜 (기본값: 오늘) |
| `status` | VARCHAR | DEFAULT '확인대기' | 납부 상태 ('확인대기', '입금완료') |

### ③ 공용 지출 내역 테이블 (`expenses`)
| 컬럼명 (Column) | 타입 (Type) | 제약 조건 | 설명 |
| :--- | :--- | :--- | :--- |
| `id` | BIGINT | PRIMARY KEY, GENERATED ALWAYS AS IDENTITY | 고유 식별자 |
| `expense_month` | VARCHAR(7) | NOT NULL | 대상 년/월 (예: '2026-05') |
| `category` | VARCHAR | NOT NULL | 지출 항목 ('공용전기세', '계단 청소', '정화조' 등) |
| `amount` | INT | NOT NULL | 지출 금액 |
| `memo` | TEXT | - | 비고 및 상세 내용 |

## 4. 핵심 기능 요구사항 & 서비스 플로우

### 📱 사용자(세대원) 플로우
1. **로그인 / 세대 선택 화면**
   * 모바일 화면에 9개 호수(B01, B02, 101, 102, 201, 202, 301, 302, 401)가 3x3 그리드 형태의 대형 버튼으로 배치됩니다.
   * 본인 호수를 터치한 후 4자리 비밀번호를 입력하여 대시보드로 진입합니다.
2. **메인 대시보드 및 입금 안내 화면**
   * **[상단] 공용 지출 내역 공유**: 관리인이 등록한 이번 달 지출 내역(항목, 금액)을 아코디언(토글) 형태로 접고 펼치며 확인할 수 있습니다.
   * **[중앙] 관리비 입금 안내**: 고정 문구("매월 관리비는 2만원입니다.")와 함께 관리인의 계좌 정보 및 텍스트 `[복사]` 버튼을 제공합니다.
   * **[중앙] 이체 날짜 캘린더**: HTML5 date input을 활용하며, 최초 진입 시 **오늘 날짜가 자동으로 입력**되어 있어야 합니다. 사용자는 필요 시에만 달력 피커를 열어 날짜를 변경합니다.
   * **[하단] 기존 입금 기록 확인**: `[입금 기록 확인]` 버튼을 누르면 과거 납부 이력 리스트(`payment_month` 역순 정렬)가 펼쳐지며, 각 내역의 상태('확인대기 🟡', '입금완료 🟢')를 실시간 조회합니다.
3. **비용 0원 입금 알림 연동 (2단계 순차 처리)**
   * 사용자가 `[입금 알림 보내기]` 버튼을 클릭하면 다음과 같이 처리됩니다.
     * **1단계(DB 저장)**: 웹 내부에서 Supabase `payments` 테이블에 데이터를 '확인대기' 상태로 즉시 인서트(Insert)합니다.
     * **2단계(문자 연동)**: 저장이 완료되면 브라우저 알림창(`alert`)을 통해 안내 후, 시스템 인터페이스(`sms:`)를 활용해 스마트폰 기본 문자 앱을 강제 실행합니다.
   * **자동 완성 문자 내용 포맷**:
     > "{호수}의 {N월} 관리비를 {캘린더날짜}에 입금했습니다. 확인 부탁드립니다."

### 👑 관리자 플로우
1. **이번 달 총괄 대시보드**
   * 9개 세대의 카드 뷰 UI가 한눈에 들어오도록 배치됩니다.
   * 당월 입금 완료 세대는 **🟢 녹색 체크**, 미입금(또는 확인대기) 세대는 **🔴 적색 경고** 표시와 함께 `[독촉]` 버튼이 노출됩니다.
   * 상단에는 당월 `공용전기세`, `계단 청소`, `계단 전등`, `정화조 비용`을 입력하고 수정할 수 있는 폼 페이지로 이동하는 버튼이 위치합니다.
2. **세대별 상세 납부 현황 및 독촉**
   * 대시보드에서 특정 호수를 클릭하면 해당 세대의 과거 매월 입금 현황 레코드가 출력됩니다.
   * `[문자 보내기]` 또는 `[독촉]` 버튼 클릭 시 관리자의 스마트폰 문자 앱이 켜지며 사전에 정의된 관리자 알림 멘트가 수신인 번호와 함께 자동 입력됩니다.
     > "안녕하세요, {호수}님. 관리비 입금 확인 부탁드립니다. {계좌번호} {은행이름} {관리인이름}"

## 5. 제약 사항 및 예외 처리 (UI/UX 룰)
* **모바일 최적화**: 모든 버튼 크기 및 입력창 폰트 스케일은 모바일 환경에서 터치 미스가 나지 않도록 최소 44px 이상의 터치 영역을 확보해야 합니다.
* **OS 호환성**: 모바일 브라우저 환경에 따라 `sms:` 프로토콜의 쿼리 스트링 매개변수가 다를 수 있으므로 JavaScript 단에서 iOS(`&body=`)와 Android(`?body=`) 기기를 판별하여 하이브리드로 대응합니다.
* **동기화 이슈 방지**: 사용자가 문자 앱 호출 후 실제 발송을 취소하더라도 DB에는 '확인대기'로 남게 되므로, 최종 검증 권한은 온전히 관리자의 '입금완료' 상태 수동 변경 기능에 의존합니다.
